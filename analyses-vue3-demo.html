📐 Ce qui diffère de la version HTML/JS actuelle
Version actuelle (bird-i18n.js)
document.addEventListener('langchange', () => {
  applyI18n(); // data-i18n statiques
  loadSpeciesDropdown();
  renderTags();
  if (selected) applyFilters(); // charts
});

Chaque page doit déclarer manuellement ce listener.
Les labels chartTitle, t('kpi_...') générés en innerHTML ne se mettent à jour que si la fonction correspondante est ré-appelée.
Version Vue 3 CDN
// Dans le template : {{ t('nav_analyses') }}
// Computed :
polarTitle: () => t('analyses_polar_title',
  { species: filters.species }),
narrative: () => buildNarr(byHour, lang),

Aucun listener manuel. Vue observe que polarTitle dépend de lang et de filters.species — dès que l'un change, le DOM se met à jour automatiquement. Même logique pour tous les labels, titres de charts, narratifs.
Coût de migration : réécrire le template HTML en directives Vue (v-for, v-if, v-model) + déplacer les fonctions async dans setup(). Les requêtes SQLite via birdQuery() restent identiques. Estimation : ~4-6h par page complexe.
